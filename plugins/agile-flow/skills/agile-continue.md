# Agile Continue - 自动任务流转

## 核心职责

实现自动任务流转：
1. 检测任务完成状态
2. 自动运行测试和验收
3. 继续下一个优先级最高的任务
4. 根据条件决定暂停或继续

---

## 执行流程

### 第一步：读取完成的任务

从 `auto_continue.flag` 读取刚完成的任务 ID。

### 第二步：运行测试

根据项目类型运行测试：
- Node.js: `npm run test:unit -- --coverage`
- Python: `pytest --cov`

### 第三步：检查验收标准

验证：
- 功能验收：功能正常工作
- 质量验收：覆盖率 ≥ 80%
- 代码验收：通过 Linting

### 第四步：更新验收报告

将测试结果记录到 `ai-docs/ACCEPTANCE.md`。

### 第五步：获取下一个任务

从状态文件读取待办任务列表，选择优先级最高的任务。

### 第六步：继续开发或暂停

- 如果有待办任务：继续开发
- 如果所有任务完成：生成迭代回顾
- 如果发现 Bug：记录到 BUGS.md 并暂停

---

## 输出结果

```
✅ 任务 TASK-001 已完成

🧪 测试结果:
- 单元测试: ✅ 全部通过
- 覆盖率: 85%
- Linting: ✅ 通过

📋 验收结果: ✅ 通过

🎯 下一个任务: TASK-002 - 实现数据导出
```

---

## 注意事项

1. 自动测试失败时尝试诊断和修复
2. 发现 Bug 时记录到 BUGS.md
3. 覆盖率低于 80% 时需要添加更多测试
